# FoodMate-AI 数据库设计文档

## 一、数据库架构总览

FoodMate-AI 采用**多数据库混合架构**，根据不同业务域的数据特点选用最合适的数据库引擎：

| 数据库                  | 引擎          | 用途                                             | 所属服务                     |
| :---------------------- | :------------ | :----------------------------------------------- | :--------------------------- |
| `food_delivery_db`      | PostgreSQL 15 | 核心业务数据（用户、商家、订单、营销、平台运营） | 用户/商家/订单/营销/平台服务 |
| `ai_pricing_db`         | PostgreSQL 15 | AI 定价分析数据（销售历史、定价提案）            | AI 定价服务                  |
| `user_profiles`（集合） | MongoDB 6.0   | 用户画像文档（偏好、行为、浏览历史）             | 用户画像服务                 |
| Redis                   | Redis 7       | 全局缓存层（热点数据毫秒级响应）                 | 全局共享                     |

### 设计原则

1. **业务数据与 AI 数据物理隔离**：`ai_pricing_db` 独立于主库，避免 AI 大量读写操作影响核心业务性能
2. **结构化数据用 PostgreSQL**：用户、订单、佣金等 Schema 固定的业务数据使用关系型数据库
3. **灵活 Schema 数据用 MongoDB**：用户画像的偏好标签、行为记录等高度灵活的数据使用文档数据库
4. **冗余换性能**：佣金记录表冗余存储服务名称和费率，避免跨表联查

## 二、主业务数据库（food_delivery_db）

主库共包含 **24 张业务表**，按功能域划分为以下六个模块。

### 2.1 用户与认证模块

#### users — 用户表

系统最核心的基础表，存储所有注册用户的认证信息和信用等级数据。

| 字段名                 | 类型         | 约束                        | 说明                              |
| :--------------------- | :----------- | :-------------------------- | :-------------------------------- |
| `id`                   | BIGSERIAL    | PRIMARY KEY                 | 自增主键                          |
| `username`             | VARCHAR(50)  | UNIQUE NOT NULL             | 用户名，唯一                      |
| `email`                | VARCHAR(100) | UNIQUE NOT NULL             | 邮箱，唯一                        |
| `password_hash`        | VARCHAR(255) | NOT NULL                    | 密码哈希值                        |
| `nickname`             | VARCHAR(50)  | —                           | 昵称                              |
| `avatar_url`           | VARCHAR(255) | —                           | 头像 URL                          |
| `phone`                | VARCHAR(20)  | —                           | 手机号                            |
| `role`                 | VARCHAR(20)  | NOT NULL DEFAULT 'customer' | 角色：customer / merchant / admin |
| `credit_level`         | INT          | DEFAULT 5                   | 信用等级（1-5，5 为最高）         |
| `recent_cancellations` | INT          | DEFAULT 0                   | 近期取消次数                      |
| `last_level_change_at` | TIMESTAMP    | —                           | 最后一次信用等级变动时间          |

#### addresses — 收货地址表

| 字段名       | 类型         | 约束                             | 说明           |
| :----------- | :----------- | :------------------------------- | :------------- |
| `id`         | BIGSERIAL    | PRIMARY KEY                      | 自增主键       |
| `user_id`    | BIGINT       | FK → users(id) ON DELETE CASCADE | 所属用户       |
| `city`       | VARCHAR(100) | NOT NULL                         | 城市           |
| `street`     | VARCHAR(255) | NOT NULL                         | 街道           |
| `detail`     | VARCHAR(255) | —                                | 详细地址       |
| `is_default` | BOOLEAN      | DEFAULT FALSE                    | 是否为默认地址 |
| `created_at` | TIMESTAMP    | DEFAULT CURRENT_TIMESTAMP        | 创建时间       |

#### cancellation_records — 取消记录表

| 字段名         | 类型      | 约束                      | 说明           |
| :------------- | :-------- | :------------------------ | :------------- |
| `id`           | BIGSERIAL | PRIMARY KEY               | 自增主键       |
