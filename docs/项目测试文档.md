# 1 测试计划

## 1.1 测试策略与目标

### 1.1.1 测试策略

本项目在软件需求分析完成后即制定测试策略，采用**测试金字塔**模型分层实施，确保从最小代码单元到完整用户场景的全方位覆盖。

**测试重点**：
- 多智能体推荐系统的核心算法正确性（Contextual Bandit 评分、MAB 策略选择、上下文信号融合）
- 订单全生命周期状态机的状态转换合法性
- 优惠券组合优化算法的计算精度和边界情况
- AI 定价流水线的事件驱动数据流正确性
- NutriVision 的图片校验、置信度路由和响应格式标准化
- 端侧 AI 与云端推荐的端云协同数据流
- JWT 安全认证与常见 Web 攻击防护

**测试难点**：
- 9 个微服务（6 Java + 3 Python）的跨语言、跨服务集成测试
- RabbitMQ 异步事件的端到端流转验证
- AI 模型输出的非确定性（LLM 生成结果的格式稳定性）
- 外部 API（和风天气、高德地图、DeepSeek、Gemini）的不可控性与降级验证

**测试分层**：

| 层次 | 测试类型 | 用例规模 | 执行条件 |
| :--- | :--- | :---: | :--- |
| 第 1 层 | 单元测试（Java / Python / Frontend） | 268 | 无需启动任何服务 |
| 第 2 层 | 集成测试 | 75 | 需 Docker 容器运行 |
| 第 3 层 | 功能测试 + 安全测试 | 41 | 需全部微服务运行 |
| 第 4 层 | 性能测试 | 267 reqs | 需全部微服务运行 |
| 第 5 层 | 验收测试 | 29 | 需全部微服务运行 |

### 1.1.2 测试目标

项目发布时的质量等级定义为：

| 质量维度 | 目标等级 | 量化标准 |
| :--- | :--- | :--- |
| 功能正确性 | A 级 | 自动化测试用例通过率 100% |
| 安全防护 | A 级 | JWT/SQL 注入/XSS 攻击全部拦截 |

| 用户体验 | B+ 级 | 核心页面加载 < 2s，AI 推荐 < 3s |
