## 环境光感知功能方案

### 一、功能定位

将手机**环境光传感器 (Ambient Light Sensor)** 的 lux 数据整合进推荐系统，作为新的上下文信号维度，影响推荐决策和 UI 展示。

### 二、核心价值

| 光线场景      | lux 范围      | 推荐策略                   | UI 策略                    |
| ------------- | ------------- | -------------------------- | -------------------------- |
| **暗光/夜间** | < 50 lux      | 偏向夜宵、热饮、暖食、酒吧 | 高对比度卡片、降低蓝光提示 |
| **室内正常**  | 50-500 lux    | 无特殊偏好                 | 标准展示                   |
| **室外明亮**  | 500-10000 lux | 偏向冰饮、清爽、外带友好   | 提高字体对比度             |
| **强烈日照**  | > 10000 lux   | 偏向冷饮、遮阳处堂食       | 户外高可见度模式           |

### 三、技术架构（遵循现有模式）

```
端侧传感器读取           端侧数据汇聚           API传输           后端决策
┌─────────────┐    ┌──────────────────┐    ┌──────────┐    ┌────────────────┐
│useAmbientLight│───▶│ useHealthContext │───▶│HomeScreen│───▶│ DecisionAgent  │
│   (新 Hook)   │    │  (扩展 lightLux  │    │ 已有逻辑  │    │ 新增第9层加分   │
│              │    │   + lightLevel)  │    │ 附带光线  │    │ ±0.15 权重     │
└─────────────┘    └──────────────────┘    └──────────┘    └────────────────┘
```

#### 4. 后端 `DecisionAgent` 新增第 9 层评分

```python
# ========== 🆕 9. 环境光线影响 ==========
light_lux = health_ctx.get("light_lux", -1)
light_level = health_ctx.get("light_level", "normal")

if light_level == "dark":
    # 暗光/夜间 → 偏向夜宵、热饮、暖食
    night_keywords = ["夜宵", "烧烤", "串", "火锅", "热饮", "奶茶", "粥", "酒吧", "小龙虾"]
    cold_keywords = ["沙拉", "冰", "冷饮", "刺身"]
    if any(k in cuisine_str for k in night_keywords):
        context_bonus += 0.15
    elif any(k in cuisine_str for k in cold_keywords):
        context_bonus -= 0.08

elif light_level in ("bright", "sunlight"):
    # 强光/户外 → 偏向冰饮、清爽
    cool_keywords = ["冰", "冷饮", "果汁", "沙拉", "轻食", "奶茶", "冰淇淋"]
    heavy_keywords = ["火锅", "烧烤", "麻辣"]
    if any(k in cuisine_str for k in cool_keywords):
        context_bonus += 0.12
    elif any(k in cuisine_str for k in heavy_keywords):
        context_bonus -= 0.06
```

### 七、风险与兜底

| 风险           | 兜底措施                                                    |
| -------------- | ----------------------------------------------------------- |
| 设备无光传感器 | `isAvailable=false` 时 lightLevel 默认 `normal`，不影响评分 |
| iOS 限制       | iOS 不直接暴露光传感器 API，通过屏幕亮度间接推算            |
| 传感器噪声     | 300ms 节流 + 移动平均平滑                                   |
| lux 值离谱     | 钳位到 0~100000 范围                                        |
