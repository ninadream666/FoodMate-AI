## 📊 平台端统计数据架构分析

### 🏗️ **数据流架构图**
```
前端页面 → Service层 → API配置 → Vite代理 → 后端微服务
   ↓           ↓          ↓           ↓          ↓
Dashboard → dashboardService → platformApi → 8088端口 → Platform Service
Orders   → orderService     → orderApi    → 8084端口 → Order Service  
Users    → userService      → userApi     → 8083端口 → User Service
Merchants→ merchantService  → merchantApi → 8081端口 → Merchant Service
```

### 📈 **各页面统计数据实现**

#### 1. **Dashboard页面 (主仪表盘)**
**文件**: `/src/pages/admin/Dashboard.jsx`
**数据源**: Platform Service (8088) 统一聚合接口
**API端点**: `/api/admin/dashboard/overview`
**实现逻辑**:
```javascript
// 获取概览数据
const overview = await dashboardService.getDashboardOverview();

// 数据映射到KPI卡片
setKpiData({
    totalSales: { amount: overview.totalRevenue || 0 },
    newUsers: { count: overview.totalUserCount || 0 },
    activeMerchants: { count: overview.activeMerchantCount || 0 },
    todayOrders: { count: overview.todayOrderCount || 0 }
});
```
**备用方案**: 当Platform Service不可用时，从各微服务聚合数据

#### 2. **订单管理页面**
**文件**: `/src/pages/admin/Orders.jsx`
**数据源**: Order Service (8084) 专用统计接口
**API端点**: `/api/admin/orders/stats`
**实现逻辑**:
```javascript
const fetchStats = async () => {
    const statsData = await orderService.getOrderStats();
    setStats({
        totalOrders: statsData.totalOrders || 0,
        pendingOrders: statsData.pendingOrders || 0,
        completedOrders: statsData.completedOrders || 0,
        totalRevenue: statsData.totalRevenue || 0
    });
};
```

#### 3. **用户管理页面**
**文件**: `/src/pages/admin/Users.jsx`

### 🔍 **统计数据为0的可能原因**

#### A. **网络层问题**
1. **后端服务未启动**: 8088, 8084, 8083, 8081端口服务状态
2. **代理配置错误**: Vite代理路径不匹配
3. **CORS问题**: 跨域请求被阻止

#### B. **认证问题** 
1. **Token过期**: adminToken无效或过期
2. **权限不足**: 管理员权限不够访问统计接口

#### C. **API接口问题**
1. **接口路径错误**: 后端API端点与前端调用不匹配
2. **数据结构变化**: 后端返回格式与前端期望不符
3. **接口未实现**: 后端统计接口还未开发完成

#### D. **数据处理问题**
1. **字段映射错误**: 前端取值字段与后端返回字段不匹配
2. **数据类型问题**: 数字字段被当作字符串处理
3. **默认值覆盖**: 异常处理时总是返回0
